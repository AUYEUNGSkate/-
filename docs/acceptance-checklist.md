## 开发验收

| 阶段 | 标准 |
|---|---|
| 项目启动 | 前端和后端都可以本机启动 |
| 健康检查 | `/api/health` 返回正常 |
| 数据库 | SQLite 文件可创建，表结构初始化成功 |
| 关键词 | 可以新增、禁用、删除关键词 |
| 来源 | 可以查看内置来源，新增和禁用来源 |
| 手动扫描 | 页面点击后能触发扫描 |
| 自动扫描 | worker 默认每 30 分钟扫描一次 |
| 热点列表 | 能展示候选热点和 AI 判别结果 |
| 未读提醒 | 新热点显示徽标和列表未读状态 |
| 标记已读 | 点击后未读数减少 |
| 设置 | 能查看扫描频率、AI 模式和 OpenRouter 配置状态 |

## OpenRouter 验收

| 项目 | 标准 |
|---|---|
| `.env` | 可以配置 `OPEN_ROUTER`，并兼容 `OPENROUTER_API_KEY` |
| Key 安全 | 前端 API 不返回完整 Key |
| 真实调用 | `AI_MODE=openrouter` 时能调用 OpenRouter |
| 结构化输出 | 返回结果符合 JSON Schema |
| 失败处理 | Key 错误、额度不足、网络失败时有错误提示和日志 |
| Mock 兜底 | `AI_MODE=mock` 时不依赖 OpenRouter 也能完成扫描 |

## 信息源验收

| 项目 | 标准 |
|---|---|
| 内置来源 | 有游戏行业和技术更新来源 |
| 自定义来源 | 页面可添加 RSS 来源 |
| 搜索来源 | 能基于关键词生成 Google News RSS 查询 |
| 时间窗口 | 默认只优先处理近 24 小时内容 |
| URL 去重 | 同一链接不重复入库 |
| 标题去重 | 高度相似标题不重复产生新热点 |
| 来源失败 | 单个来源失败不影响整体扫描 |

## UI 验收

| 项目 | 标准 |
|---|---|
| 响应式 | 桌面和移动端均可使用 |
| 独特性 | 页面有游戏情报雷达风格，不像通用后台模板 |
| 可操作性 | 首屏能直接看到热点、未读数和扫描入口 |
| 可读性 | 标题、评分、来源、时间、AI 理由清晰 |
| 状态反馈 | 扫描中、扫描成功、扫描失败都有反馈 |
| 无重叠 | 移动端和桌面端文字不互相遮挡 |

## 测试命令

开发完成后至少运行：

```bash
npm test
npm run build
```

Windows 环境建议优先使用 Git Bash 运行上面的命令，避免 PowerShell 执行策略拦截 `npm.ps1`。如果必须使用 PowerShell，请改用：

```powershell
& 'D:\Node.js\npm.cmd' test
& 'D:\Node.js\npm.cmd' run build
```

如启动本地服务，应提供本地访问地址，例如：

```txt
http://localhost:5173
```

## 人工验收流程

1. 配置 `.env` 中的 OpenRouter Key。
2. 启动前端、后端和 worker。
3. 打开网页。
4. 添加关键词：
   - `AI 编程`
   - `Unity`
   - `游戏出海`
5. 点击立即扫描。
6. 查看热点列表是否出现候选内容。
7. 打开热点详情，确认 AI 评分和理由存在。
8. 确认未读徽标出现。
9. 将热点标记为已读，确认未读数减少。
10. 修改扫描频率，确认设置保存。
11. 等待或手动触发下一次扫描，确认不会重复产生同一热点。
