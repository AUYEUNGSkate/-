# 游戏热点雷达需求文档

## 背景

作为游戏行业从业者，用户需要第一时间获取游戏行业热点，包括行业动态、厂商/产品变化、技术更新、AI 编程与生产力工具变化等。用户不希望依赖人工手动搜索，而是希望用一个轻量工具自动发现指定关键词或范围内的热点变化，并在站内及时提示。

项目定位为个人轻量工具，采用敏捷开发方式。第一阶段优先完成可用网页版，确认功能正常后，再封装为 Agent Skill，交给其他 AI 执行监控和发现热点。

## 目标用户

- 游戏行业从业者
- 关注游戏行业动态、技术更新、厂商变化、AI 工具变化的人
- 希望低成本、本机运行、快速获得热点线索的人

## 核心目标

1. 用户可以手动输入需要监控的关键词。
2. 系统定时搜集指定范围内的热点内容。
3. 系统使用 AI 判断内容是否真实相关、是否疑似假冒或标题党。
4. 用户可以在响应式 Web 页面中查看新热点、历史热点、AI 判断理由和来源。
5. Web 版验收后，将能力封装为 Agent Skill。

## 已确认决策

| 维度 | 决策 |
|---|---|
| 产品形态 | 响应式 Web 页面 |
| 开发顺序 | 先 Web 版，验收后再 Agent Skill |
| 信息源范围 | 游戏行业 + 技术更新 |
| 来源配置 | 内置来源 + 页面可编辑 |
| 通知方式 | 只做站内通知 |
| 站内通知形式 | 徽标 + 列表未读状态 |
| 检查频率 | 默认每 30 分钟 |
| AI 服务 | OpenRouter |
| API Key 状态 | 用户已获得 OpenRouter API Key |
| v1 部署方式 | 本机运行 |

## v1 范围

| 功能 | v1 要求 |
|---|---|
| 关键词管理 | 新增、启用/禁用、删除关键词 |
| 热点范围管理 | 支持输入如“AI 编程”“游戏出海”“Unity”等范围 |
| 信息源管理 | 预置游戏行业与技术信息源，支持页面编辑 |
| 手动扫描 | 页面点击后立即执行一次扫描 |
| 自动扫描 | worker 每 30 分钟自动扫描 |
| 热点列表 | 展示标题、来源、发布时间、关键词、评分、AI 理由、链接 |
| 未读提醒 | 新热点显示未读徽标和列表高亮 |
| 已读管理 | 支持将热点标记为已读 |
| AI 判别 | 使用 OpenRouter 输出结构化判断 |
| Mock 兜底 | OpenRouter 不可用时可切换 Mock 模式 |

## v1 不做

| 不做项 | 原因 |
|---|---|
| Telegram / Webhook / 浏览器通知 | 本轮已确认只做站内通知 |
| 登录系统 | v1 是个人本机工具 |
| 多用户权限 | 暂无多人协作需求 |
| 云端部署 | 先保证本机可用 |
| 移动 App | 响应式网页即可 |
| 复杂推荐算法 | v1 以规则 + AI 评分为主 |

## 成功标准

1. 用户可以在页面添加关键词和热点范围。
2. 用户可以查看内置信息源并编辑来源。
3. 用户可以手动扫描，并看到候选热点进入列表。
4. worker 可以按 30 分钟频率自动扫描。
5. 每条候选热点都有 AI 结构化判别结果。
6. 新热点会出现在未读徽标和未读列表中。
7. 用户点击热点后可以查看 AI 理由和原始链接。
8. API Key 不出现在前端或仓库中。
9. Web 版通过验收后，再开始 Agent Skill。
